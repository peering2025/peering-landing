# /peering-news — 피어링 SEO 콘텐츠 자동 생성·배포 스킬

## 개요

이 스킬은 특수교사를 위한 SEO 최적화 뉴스 콘텐츠를 한 번의 명령으로 생성·배포합니다.
생성된 글은 /news 리스트(Featured + Grid)와 /news/[slug] 상세 페이지에 즉시 반영됩니다.

---

## 실행 절차

### Step 1 — 주제 분석

인수($ARGUMENTS)가 있는 경우: 전달된 키워드를 중심으로 제목, 슬러그, 콘텐츠 방향을 기획한다.
인수가 없는 경우: scripts/generate-seo-content.mjs의 TOPIC_POOL을 Read로 읽어온다.
src/content/news.ts의 기존 slug/title 목록과 대조하여 미개척 주제 하나를 선정한다.

주제 선정 기준:
- 네이버에서 구체적인 검색 수요가 있는 롱테일 키워드
- 기존 글과 주제가 겹치지 않을 것
- 특수교사의 실제 업무 페인 포인트(시간표, 시수, IEP, 협업, 통합교육 등)

---

### Step 2 — 콘텐츠 생성

src/content/news.ts의 NewsPost 타입을 정확히 준수하여 객체를 생성한다.

  slug      영소문자-하이픈만, 30자 이내
  title     40자 이내, 특수교육/특수학급/피어링 중 1개 이상 포함
  date      YYYY-MM-DD (오늘 날짜)
  category  팁 & 노하우 | 업데이트 | 공지사항 | 해외 연구 동향
  excerpt   80자 이내 — 네이버 SEO 규칙 엄수, 핵심 키워드와 구체적 내용 자연 포함
  readTime  3~7 사이 정수
  image     아래 이미지 배정 기준 참고
  content   풍성한 HTML (최소 2,000자, h2/h3/p/ul/li/strong 체계적 활용)

---

#### content 품질 기준 — 반드시 준수

**글 길이 및 구조:**
- 전체 분량: 최소 2,000자 (공백 포함 HTML 기준), 권장 3,000자 이상
- readTime: 분량에 비례하여 산정 (600자=5분, 1200자=10분, 1800자=14분 기준)
- h2 섹션: 최소 4개 이상
- h3 서브섹션: 주요 h2마다 2~3개 이상

**도입부 (첫 h2 바로 아래 첫 p):**
- 단순 정의 나열 금지
- 독자를 끌어당기는 문제 제기, 반전 서술, 또는 핵심 인사이트로 시작
- 예: "~라고 생각하면 오산입니다", "~가 달라진다", "~의 진짜 이유는"

**본문 각 섹션:**
- 섹션당 p 태그 최소 2개 이상 (1문단 1섹션 금지)
- 단순 나열 금지 — 각 항목에 이유·맥락·예시를 함께 제시
- 해외 사례/연구 언급 시: 기관명, 연도, 구체적 수치(%) 또는 결과를 명시
- ul/li 항목: 단순 키워드 나열 금지, 반드시 bold 리드 + 실천 방법 또는 이유 설명

**ul/li 작성 원칙:**
  나쁜 예: <li>개별화교육이 중요합니다.</li>
  좋은 예: <li><strong>IEP 목표를 수업 목표와 연결하라:</strong> 매 수업 시작 전 IEP의 단기 목표 1개를 오늘의 학습 목표로 명시하면, 목표 달성률이 평균 18%p 향상됩니다.</li>

**summary-box (핵심 인사이트 박스):**
- 항목 수: 최소 5개
- 각 항목: bold 리드 문장 + 구체적 실행 방법 또는 근거 포함
- 마지막 항목은 측정·기록·데이터 관련 내용으로 마무리

**마지막 단락 (피어링 언급):**
- 홍보성 문장 1개로 끝내지 말 것
- 앞 본문의 특정 키워드 또는 주제와 자연스럽게 연결하여 2~3문장으로 작성

content 작성 규칙:
- h2로 시작, h3으로 섹션 구분
- p, ul, li, strong 적극 활용
- 결론 단락에 피어링 솔루션 자연스럽게 언급
- 전문적이고 정중한 특수교사 어조 유지
- 백틱 문자 절대 사용 금지 — 강조는 strong 태그로 대체

---

### Step 3 — 이미지 배정

scripts/generate-seo-content.mjs의 IMAGE_POOL에서 키워드로 카테고리를 판단하여 URL 선택.

  업데이트·출시·기능·버전       → technology 풀
  협업·공동편집·팀·실무사·배정  → collaboration 풀
  시간표·일정·계획·시수·편성    → planning 풀
  수업·학급·교실·학생·장애      → classroom 풀
  교사·강의·안내·공지           → teaching 풀
  그 외                         → study 풀

이미지는 /news Featured 카드(전체 너비 aspect-video), Grid 카드 썸네일,
상세 페이지 히어로, Open Graph 이미지로 동일하게 활용된다.

---

### Step 4 — 코드 반영

1. src/content/news.ts를 Read 도구로 읽는다.
2. "export const newsPosts: NewsPost[] = [" 마커를 찾는다.
3. 마커 바로 뒤에 새 객체를 첫 번째 요소로 삽입한다.
4. Write 도구로 전체 파일을 저장한다.

주의: 기존 글 데이터를 절대 삭제·수정하지 말 것.
content 필드는 템플릿 리터럴로 감싸고 .trim() 적용.

빌드 실패 시 오류를 수정하고 재빌드하여 반드시 성공을 확인한다.

## 참고 — 좋은 글 vs 나쁜 글 예시

### 나쁜 글 패턴 (절대 금지)
```
<h3>미국의 UDL 적용 사례</h3>
<p>미국에서 UDL을 적용한 여러 사례가 있습니다. 예를 들어, 메사추세츠 주의 한 초등학교에서는
모든 교실에서 UDL 원칙을 적용하여 학습 환경을 조성했습니다. 학교는 학생의 선택권을 확대하고,
다양한 학습 자료를 제공하여 학습 참여를 증진시켰습니다.</p>
```
문제: 섹션당 p 1개, 기관 맥락 없음, 수치 없음, 인사이트 없음.

### 좋은 글 패턴 (이 수준으로 작성)
```
<h3>미국 — 통합학급에서의 UDL 시스템적 도입</h3>
<p>매사추세츠주 보스턴 공립학교 지구(BPS)는 2018년부터 지구 전체에 UDL을 단계적으로 도입해왔습니다.
단순히 개별 교사의 수업 방식을 바꾸는 것이 아니라, 교육과정 설계·평가 도구·교사 연수·학교 환경
전반을 UDL 프레임으로 재구조화했습니다.</p>
<p>핵심은 <strong>'유연한 평가(Flexible Assessment)'</strong>였습니다. 3년 후 추적 결과,
특수교육 대상 학생의 일반학급 참여율이 62%에서 79%로 증가했고, 교사들은 "수업 준비 방식 자체가
바뀌었다"고 보고했습니다.</p>
```
강점: 구체적 기관·연도, 변화의 메커니즘 설명, 수치 근거, 당사자 인용.
